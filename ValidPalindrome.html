Finding Valid Palindrome

Introduction

A valid palindrome is a string that reads the same backward as forward, ignoring case, punctuation, spaces, and other non-alphanumeric characters. In other words, when the string is processed to remove all non-alphanumeric characters and converted to the same case, it should look the same when reversed.
The Two-Pointer technique is an efficient way to determine if a string is a palindrome. This method involves using two pointers to compare characters from the beginning and end of the string, moving towards the center.

Step By Step Explanation

To check if a string is a valid palindrome, one typically follows these steps:

  1. Helper Function: is_alphanumeric(c) checks if a character is alphanumeric using c.isalnum().
  2. Pointer Initialization: left starts at the beginning (0) and right at the end (len(s) - 1) of the string.
  3. Character Skipping: The inner while loops ensure that left and right skip over non-alphanumeric characters.
  4. Character Comparison: The characters at the left and right pointers are compared after converting them to lowercase.
  5. Pointer Movement: The pointers are moved towards each other if the characters match.
  6. Early Termination: If a mismatch is found, the function immediately returns False.
  7. Completion Check: If the pointers cross each other without finding a mismatch, the function returns True, indicating the string is a valid palindrome.

This approach ensures the algorithm runs in linear time, O(n), where n is the length of the string, making it efficient.

Python Code

                
def is_valid_palindrome(s):
    # Helper function to determine if a character is alphanumeric
    def is_alphanumeric(c):
        return c.isalnum()

    # Initialize pointers
    left, right = 0, len(s) - 1

    while left < right:
        # Move left pointer to the next alphanumeric character
        while left < right and not is_alphanumeric(s[left]):
            left += 1

        # Move right pointer to the previous alphanumeric character
        while left < right and not is_alphanumeric(s[right]):
            right -= 1

        # Compare characters at left and right pointers
        if s[left].lower() != s[right].lower():
            return False

        # Move pointers towards each other
        left += 1
        right -= 1

    return True

# Example usage
s1 = "A man, a plan, a canal: Panama"
print(is_valid_palindrome(s1)) # Output: True

s2 = "race a car"
print(is_valid_palindrome(s2)) # Output: False

Example

This is where the pointers will be the very first time we enter our while loop. It will see that 9 is equal to 9, so it will continue through with incrementing our left pointer and decrementing our right pointer.

This is the second iteration through the while loop. Each pointer has moved and now 8 is being checked versus 8. We will continue through our loop.

This is the third time through the loop and the pointers are now at 7 & 7. This will not hit our conditional inside the loop and will keep going.

At this point, the left and right pointers are now equal to each other and the condition of the while loop has been met to break out of the loop. We can now return true.

The two pointer technique can lead you to a solution that has an efficient time complexity. This solution has an O(N) time complexity. In the worst case scenario, which is that an integer is in fact a palindrome, we only have to loop through half the length of our integer.
